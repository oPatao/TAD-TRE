# TAD-TRE
Relatório do codigo da TP01
O projeto foi estruturado seguindo os princípios de modularidade exigidos para a implementação de um Tipo Abstrato de Dados (TAD). O sistema foi particionado em três arquivos: tratamento.h (definição das interfaces, structs e cabeçalhos), tratamento.c (implementação da regra de negócio e parse) e main.c (orquestração do fluxo de execução).
Estrutura de dados e gerenciamento de memória Para suportar o carregamento da base de dados completa sem conhecimento prévio do número total de linhas, a abordagem escolhida foi a implementação de uma Lista Encadeada Simples Dinâmica. A base do TAD é a struct Unidade, que mapeia as colunas de texto em arrays de char e consolida as 23 colunas numéricas em um único vetor float dados[23]. Isso otimizou o acesso aos dados via índices durante as operações matemáticas. Cada novo registro gera uma alocação dinâmica (malloc) de um Nodo que é inserido no final da lista.
Para evitar a abertura manual de 27 arquivos, o ‘main.c’ possui um array estático com as siglas dos estados, utilizando um laço for para montar o nome do arquivo dinamicamente (sprintf) e realizar a leitura em lote. O parse de cada linha lida pelo fgets é feito utilizando a função strtok (separando pela vírgula). Durante esse processo, o código realiza a higienização das strings (removendo aspas duplas) e converte os valores contábeis de string para float utilizando atof, populando o TAD em memória com a base nacional consolidada.
Item 2: O processamento das metas foi centralizado na função gerar_resumo_tribunais(). A função itera sobre a lista completa para cada uma das siglas de tribunal. Durante a varredura, variáveis acumuladoras registram o somatório de casos_novos, julgados e as variáveis base das metas (distribuídos, suspensos, etc.). Após o somatório, as fórmulas das Metas 1, 2A, 2Ant, 4A e 4B são aplicadas condicionalmente, blindando o sistema contra exceções como o caso que divisor termine em zero. Os resultados são consolidados e exportados diretamente para o arquivo resumo_tribunais.csv.
Item 3: A funcionalidade de extração por município pede a entrada do usuário via scanf(" %99[^\n]"), pra ter certeza que vai funcionar  a leitura de cidades com nomes compostos e prevenindo o consumo indevido de lixo de memória do buffer (\n). Para evitar falhas na busca por divergência de caixa alta/baixa, a string de entrada é submetida a um loop com a função toupper (biblioteca <ctype.h>), padronizando o input. A varredura na lista utiliza strcasecmp para encontrar a correspondência. Ao encontrar os nós que fazem sentido pro sistema, o sistema reconstrói a linha completa incluindo as colunas, e grava no arquivo de saída nomeado dinamicamente ({MUNICIPIO}.csv).
no fim da execução do programa, mas antes de encerrar o programa, a função liberar_lista() é invocada para percorrer o TAD executando a rotina de free() em cada nó, garantindo a devolução completa da memória ao sistema operacional e prevenindo vazamentos.
